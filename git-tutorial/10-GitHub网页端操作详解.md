## 10.3 Fork 仓库

Fork 是将别人的仓库复制到你的账号下。

### 操作步骤

**Step 1：找到要 Fork 的仓库**

访问任意开源项目，例如：`https://github.com/facebook/react`

**Step 2：点击 Fork 按钮**

```
页面右上角
┌─────────┐
│  🍴 Fork │  ← 点击这个按钮
└─────────┘
```

**Step 3：配置 Fork**

```
┌──────────────────────────────────────────────────────────────┐
│ Create a new fork                                             │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│ Owner *                                                       │
│ ┌─────────────────────────┐                                  │
│ │ ▼ 你的用户名             │  ← 选择你的账号                   │
│ └─────────────────────────┘                                  │
│                                                               │
│ Repository name *                                             │
│ ┌─────────────────────────────────────────────────────────┐  │
│ │ react                    ← 默认和原仓库同名，可以修改     │  │
│ └─────────────────────────────────────────────────────────┘  │
│                                                               │
│ ☑️ Copy the main branch only   ← 只复制 main 分支（推荐）    │
│                                                               │
│              ┌─────────────────────────────┐                  │
│              │      Create fork            │  ← 点击创建      │
│              └─────────────────────────────┘                  │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

**Step 4：Fork 完成**

```
你的仓库地址：https://github.com/你的用户名/react

页面会显示：
"forked from facebook/react"
```

## 10.4 创建 Pull Request（网页端详细步骤）

### 场景设定

假设你已经推送了分支 `feature-login` 到远程，现在要在网页上创建 PR。

### 方法1：通过推送后的提示创建

推送代码后，GitHub 会自动显示提示：

```
┌──────────────────────────────────────────────────────────────┐
│ 你的分支 feature-login 已推送                                 │
│                                                               │
│ ┌─────────────────────────────────────────────────────────┐  │
│ │    Compare & pull request                               │  │  ← 点击这个绿色按钮
│ └─────────────────────────────────────────────────────────┘  │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

### 方法2：手动创建 PR

**Step 1：进入 Pull requests 页面**

```
点击顶部导航栏的 [🔀 Pull requests]
```

**Step 2：点击新建按钮**

```
页面右侧绿色按钮
┌─────────────────────────┐
│  New pull request       │  ← 点击
└─────────────────────────┘
```

**Step 3：选择分支**

```
┌──────────────────────────────────────────────────────────────┐
│ Compare changes                                               │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│ base: main  ◄──  compare: feature-login                       │
│      ▼                    ▼                                   │
│ ┌───────────┐           ┌───────────────┐                    │
│ │ main      │           │ feature-login │  ← 选择要合并的分支  │
│ └───────────┘           └───────────────┘                    │

## 10.6 合并 Pull Request

当审查通过后，可以合并 PR：

```
页面底部
┌──────────────────────────────────────────────────────────────┐
│ Merging is blocked                                             │
│ The base branch restricts merging to authorized users.        │
│ ───────────────────────────────────────────────────────────── │
│                                                              │
│ ▼ Merge pull request                                          │
│                                                              │
│   ○ Create a merge commit                                     │
│     <所有提交都会被保留，并创建一个合并提交>                     │
│                                                              │
│   ○ Squash and merge                                          │
│     <所有提交压缩为一个提交>                                   │
│                                                              │
│   ○ Rebase and merge                                          │
│     <重新应用在主分支上，线性历史>                             │
│                                                              │
│ Head repository: 你的用户名/website compare: feature-login    │
│ Base repository: company/website  compare: main               │
│                                                              │
│ [Confirm merge]  [Cancel]                                     │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

**合并成功后显示：**

```
┌──────────────────────────────────────────────────────────────┐
│ ✅ Pull request successfully merged and closed                 │
│                                                              │
│ feat(login): 实现用户登录功能 was merged into main.            │
│                                                              │
│ [Delete branch]  ← 点击删除已合并的分支                       │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```
