# 第4节：分支管理

## 📚 本节目标
- 理解分支的概念
- 创建、切换、合并分支
- 解决分支冲突

---

## 4.1 什么是分支？

分支就像是代码的平行宇宙，让你可以在不影响主代码的情况下开发新功能。

```
        main
          │
    A ─── B ─── C ─── D
                    │
                  feature
```

### 为什么使用分支？

- 🆕 开发新功能，不影响稳定版本
- 🐛 修复 bug，隔离测试
- 👥 多人协作，各自开发
- 🧪 实验性改动，随时可以丢弃

---

## 4.2 查看分支

```bash
# 查看所有分支
git branch

# 查看所有分支（包括远程分支）
git branch -a

# 查看分支详细信息
git branch -v
```

### ✅ 预期输出

```
* main
```

> 💡 `*` 表示当前所在的分支

---

## 4.3 创建和切换分支

### 创建新分支

```bash
# 创建新分支
git branch feature-login

# 查看分支列表
git branch
```

### ✅ 预期输出

```
  feature-login
* main
```

### 切换分支

```bash
# 切换到 feature-login 分支
git checkout feature-login

# 或者使用 switch（Git 2.23+ 推荐）
git switch feature-login
```

### ✅ 预期输出

```
Switched to branch 'feature-login'
```

### 查看当前状态

```bash
git branch
```

### ✅ 预期输出

```
* feature-login
  main
```

### 创建并切换（一步完成）

```bash
# 创建并立即切换
git checkout -b feature-register

# 或者
git switch -c feature-register
```

---

## 4.4 在分支上工作

### 在 feature-login 分支上添加功能

```bash
# 确保在 feature-login 分支上
git switch feature-login

# 创建登录功能文件
echo "function login() { console.log('用户登录'); }" > login.js
git add login.js
git commit -m "添加登录功能"

# 查看日志
git log --oneline
```

### ✅ 预期输出

```
abc1234 添加登录功能
...之前的提交...
```

### 切换回 main 分支查看

```bash
git switch main
ls
```

### ✅ 预期输出

```
# 没有 login.js 文件！
readme.txt  version.txt
```

> 💡 main 分支上没有 login.js，因为我们在 feature-login 分支上创建的它

---

## 4.5 合并分支

### 将 feature-login 合并到 main

```bash
# 确保在 main 分支上
git switch main

# 合并 feature-login 分支
git merge feature-login
```

### ✅ 预期输出（快进合并）

```
Updating def5678..abc1234
Fast-forward
 login.js | 1 +
 1 file changed, 1 insertion(+)
 create mode 100644 login.js
```

### 查看结果

```bash
ls
cat login.js
```

### ✅ 预期输出

```
function login() { console.log('用户登录'); }
```

---

## 4.6 非快进合并

让我们创建一个需要真正合并的场景：

```bash
# 在 main 上创建一个新分支
git checkout -b feature-profile

# 修改 login.js
echo "function getProfile() { return {}; }" >> login.js
git add login.js
git commit -m "添加获取用户信息功能"

# 切换回 main
git switch main

# 在 main 上也修改 login.js
echo "// 主分支的修改" >> login.js
git add login.js
git commit -m "主分支修改 login.js"
```

### 合并（会产生冲突）

```bash
git merge feature-profile
```

### ✅ 预期输出（有冲突）

```
Auto-merging login.js
CONFLICT (content): Merge conflict in login.js
Automatic merge failed; fix conflicts and then commit the result.
```

---

## 4.7 解决冲突

### 查看冲突文件

```bash
cat login.js
```

### ✅ 预期输出

```javascript
function login() { console.log('用户登录'); }
<<<<<<< HEAD
// 主分支的修改
=======
function getProfile() { return {}; }
>>>>>>> feature-profile
```

### 冲突标记说明

```
<<<<<<< HEAD      # 当前分支（main）的内容
// 主分支的修改
=======           # 分隔线
function getProfile() { return {}; }
>>>>>>> feature-profile  # 合并分支的内容
```

### 手动解决冲突

编辑 `login.js`，保留需要的内容：

```javascript
function login() { console.log('用户登录'); }
// 主分支的修改
function getProfile() { return {}; }
```

### 提交解决后的文件

```bash
# 添加解决后的文件
git add login.js

# 完成合并提交
git commit -m "合并 feature-profile 分支，解决冲突"
```

---

## 4.8 删除分支

```bash
# 删除已合并的分支
git branch -d feature-login
git branch -d feature-profile

# 强制删除未合并的分支（慎用）
git branch -D feature-xxx
```

## 4.9 分支管理最佳实践

```
main          ●────────●────────●────────●────────●
              ↑                                    ↑
           稳定版本                              生产环境
              │                                    │
 develop      └────────●────────●────────●────────┘
                        ↑                  ↑
                     开发中的功能        开发中的功能
                        │                  │
 feature/login          ●────────●         │
                                          │
 feature/register                         ●────────●
```

### 常用分支策略

| 分支名 | 用途 |
|--------|------|
| `main` | 稳定版本，随时可以发布 |
| `develop` | 开发分支，日常开发 |
| `feature/xxx` | 新功能开发 |
| `bugfix/xxx` | Bug 修复 |
| `hotfix/xxx` | 紧急修复 |

## 4.10 常用命令速查表

| 命令 | 作用 |
|------|------|
| `git branch` | 查看分支 |
| `git branch <name>` | 创建分支 |
| `git checkout <name>` | 切换分支 |
| `git switch <name>` | 切换分支（新命令） |
| `git checkout -b <name>` | 创建并切换 |
| `git switch -c <name>` | 创建并切换（新命令） |
| `git merge <name>` | 合并分支 |
| `git branch -d <name>` | 删除分支 |
