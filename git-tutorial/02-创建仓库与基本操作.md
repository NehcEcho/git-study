## 2.1 Git 工作区域

在开始之前，先理解 Git 的三个工作区域：

```
┌─────────────────────────────────────────────────────────────┐
│                      Git 工作区域                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────┐      ┌─────────────┐      ┌───────────┐  │
│   │  工作区     │  →   │   暂存区    │  →   │  版本库   │  │
│   │ (Working    │ add  │   (Stage/   │commit│ (Repository│ │
│   │  Directory) │      │   Index)    │      │           │  │
│   └─────────────┘      └─────────────┘      └───────────┘  │
│        ↑                              ↑                    │
│        └──────── checkout ────────────┘                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

- **工作区**：你实际编辑文件的目录
- **暂存区**：临时存放即将提交的文件
- **版本库**：Git 存储所有版本历史的地方

---

## 2.2 创建仓库

有两种方式创建 Git 仓库：

### 方式1：初始化新仓库

```bash
# 1. 创建一个项目目录
mkdir my-project
cd my-project

# 2. 初始化 Git 仓库
git init
```

### ✅ 预期输出

```
Initialized empty Git repository in C:/Users/XXX/my-project/.git/
```

### 方式2：克隆现有仓库（下一节详细讲）

```bash
git clone <仓库地址>
```

## 2.3 查看仓库状态

```bash
# 查看当前仓库状态
git status
```

### ✅ 预期输出（空仓库）

```
On branch main

No commits yet

nothing to commit (create/copy files and use "git add" to track)
```

---

## 2.4 添加文件到暂存区

### 创建示例文件

```bash
# 创建一个简单的文件
echo "Hello, Git!" > readme.txt

# 查看文件内容
cat readme.txt
```

### 查看状态

```bash
git status
```

### ✅ 预期输出

```
On branch main

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        readme.txt

nothing added to commit but untracked files present (use "git add" to track)
```

> 💡 `Untracked files` 表示文件未被 Git 跟踪

### 添加到暂存区

```bash
# 添加单个文件
git add readme.txt

# 添加所有文件（常用）
git add .

# 添加所有 .txt 文件
git add *.txt
```

### 再次查看状态

```bash
git status
```

### ✅ 预期输出

```
On branch main

No commits yet

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)
        new file:   readme.txt
```

> 💡 `Changes to be committed` 表示文件已在暂存区，等待提交

---

## 2.5 提交到版本库

```bash
# 提交暂存区的文件到版本库
git commit -m "首次提交：添加 readme.txt"
```

### ✅ 预期输出

```
[main (root-commit) a1b2c3d] 首次提交：添加 readme.txt
 1 file changed, 1 insertion(+)
 create mode 100644 readme.txt
```

> 💡 `-m` 参数用于添加提交说明，描述这次提交做了什么

### 提交后的状态

```bash
git status
```

### ✅ 预期输出

```
On branch main
nothing to commit, working tree clean
```

---

## 2.6 修改文件并再次提交

### 修改文件

```bash
# 修改文件内容
echo "这是我的第一个 Git 项目" >> readme.txt

# 查看文件内容
cat readme.txt
```

### ✅ 文件内容

```
Hello, Git!
这是我的第一个 Git 项目
```

### 查看状态

```bash
git status
```

### ✅ 预期输出

```
On branch main
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git restore <file>..." to discard changes in working directory)
        modified:   readme.txt

no changes added to commit (use "git add" to add)
```

> 💡 `Changes not staged for commit` 表示文件被修改但尚未添加到暂存区

### 查看具体修改内容

```bash
# 查看工作区与暂存区的差异
git diff
```

### ✅ 预期输出

```diff
diff --git a/readme.txt b/readme.txt
index 2f4d0c7..8e27be7 100644
--- a/readme.txt
+++ b/readme.txt
@@ -1 +1,2 @@
 Hello, Git!
+这是我的第一个 Git 项目
```

### 添加并提交

```bash
# 添加修改到暂存区
git add readme.txt

# 提交
git commit -m "修改 readme.txt：添加项目描述"
```

## 2.7 常用命令速查表

| 命令 | 作用 |
|------|------|
| `git init` | 初始化仓库 |
| `git status` | 查看仓库状态 |
| `git add <file>` | 添加文件到暂存区 |
| `git add .` | 添加所有修改到暂存区 |
| `git commit -m "消息"` | 提交到版本库 |
| `git diff` | 查看修改内容 |
